# Overview

When working on tasks or completing plans, follow these guidelines to ensure alignment on what the project is, the scope, and ensuring features are thought through completely and deliver a polished and fully-featured user experience.

## What the project is

This app is a church service presentation solution that at its core is a clone of ProPresenter that has some improvements and custom integrations with other apps from the church development team making this app.

## Implementation considerations

When developing something, it is critical to understand how the feature will interact with all the other systems and functions of the app and build with them in mind and ensure everything will work together as requested.

If the task involves building a feature from ProPresenter, and you don't know the feature's details, request clarification before proceeding.

It is cruital that you reason how every new feature or change will impact the app, and take steps and actions to ensure you know what you are touching and inform the user what you might be changing in a particular system or feature. You need to plan to-do items for scanning for broken, leftover, or unfinished features around what you are working on, and if you are confident you can fix them with out input or clarification, fix or remove the code. But if you need guidence, request the user's input before proceeding.
